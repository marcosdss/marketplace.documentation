# Spike - LIONS 123

### Checkout | Padronizar campos de preenchimento obrigatório

**1. Introdução**

Desejamos que o cliente veja todos os campos obrigatórios de forma mais clara no Checkout.

**2. Solução**

**2.1** Podemos utilizar o método **blur** ou **focusout** para verificar se os campos estão devidamente preenchidos. Com isso, sinalizamos o usuário antes de prosseguir.

**2.2** Utilizar a classe **disabled** nos botões de prosseguimento, caso os campos não estejam preenchidos.

**2.3** Etapa de pagamento é um **iframe**, então **não é possível inserir script**. Já existe uma validação nativa mas está sem o texto **campo obrigatório** (oculto via CSS). Se isso ajuda, podemos exibi-lo.

:::info
Alguns campos já possuem essa dinâmica de forma nativa (Dados Pessoais, por exemplo) e funcionam corretamente. Campo de **número** e **destinatário** nem sempre funcionam.
:::

**3. Diretrizes para o Desenvolvimento**

:::warning
Essa modificação pode sofrer alguma instabilidade pois se trata de um ambiente totalmente construido pela VTEX. Cada dinâmica nova é inserida fora do escopo do código fonte.
:::

**3.1 Criar uma função**

Criar uma função com os métodos sugeridos acima, seguindo o escopo do projeto, dentro das respectivas páginas.

**3.2 Chamada de função**

A chama deve ser feita dentro de **customFields()**.

**3.3 Testes de estresse**

Se possível, simular testes em conexão mais lenta.
