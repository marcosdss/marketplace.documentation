# Spike - LIONS 333

### My account | Levantamento de melhorias técnicas

**1. Introdução**

Desejamos que tenhamos autonomia para alterar nomes dos itens no my account e para alterar call to actions.

**2. Solução**

**ANALISE GERAL**

O My Account não é aberto para modificações assim como o Checkout, portanto, não há como fazer modificações de texto ou nas dinâmicas da página. Só há possibilidade de modificações visuais (cores, fontes, ocultar elementos e ícones) e ainda com certas limitações.

Sem acesso ao código nos força a dois caminhos, utilizar CSS para forçar algum efeito visual ou um planjear um desenvolvimento do zero.

:::warning
Um desenvolvimento do zero requer estudos e analise de viabilidade junto a VTEX. Questões de segurança, API' privadas, acessos a números de cartão do cliente, etc.
:::

**MODIFICAÇÕES DE TEXTO**

Segue os locais de alteração com base na descrição da [SPIKE](https://redeoba.atlassian.net/browse/LIONS-333).

1. Nome dos menus: Possível via CSS; pseudo-elemento;
2. Mensagens dos pop up's: Não há possibilidade. A VTEX não faz diferenciação dos nomes (classes do CSS) dos pop up's, portanto, não há como mudar e se houvesse diferença o texto seria mudado via CSS (o que não é o ideal). Vale reforçar que não há como criar um novo pop up pois não temos acesso ao código para criar um novo disparo de pop up.
3. Call to actions dos botões: Possível via CSS; pseudo-elemento;

**3. Diretrizes para o Desenvolvimento**

As mudanças de texto seriam via CSS como já ocorre atualmente, utilizando pseudo-elementos, after & before.
